# Week 6 Assignments (Day 15–17)

Objective  
To practice React state management, backend integration, and global state sharing using Context API.

Work Description  

Day 15  
- Built a User App with React.  
- Created `App` component using `useState` to manage user count.  
- Implemented `increment` function to update count when a button is clicked.  
- Built `UserCount` component to display total users added.  
- Built `Users` component to render a list of users.  
- Built `UserCard` component with a button to add users.  
- Styled components using Tailwind CSS.  

Day 16  
- Implemented backend for Employee Management App.  
- Created routes for:  
  - **Create Employee** → add new employee details.  
  - **Read All Employees** → fetch and display all employee records.  
- Connected backend with frontend to allow user interaction.  
- Tested API endpoints to ensure proper data flow between client and server.  

Day 17  
- Implemented Context API to share state across the application.  
- Created a counter state and provided it through Context.  
- Used `Context.Provider` and `useContext` hooks to access and update state.  
- Ensured multiple components could access and update the counter.  
- Kept the counter synchronized across the app whenever updated.  

Outcome  
- Learned how to manage state in React using `useState`.  
- Practiced passing props to child components for event handling.  
- Strengthened knowledge of backend routes and frontend integration.  
- Gained experience in building full‑stack applications with consistent data flow.  
- Improved ability to share and synchronize state globally using Context API.
